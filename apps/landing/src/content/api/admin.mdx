# Admin API Reference

The Admin API allows you to manage API keys, organizations, teams, projects, and provider credentials programmatically.

<Callout type="warning">
Admin endpoints require authentication with an admin-scoped API key or the `AURA_ADMIN_KEY`.
</Callout>

---

## Authentication

Admin endpoints require the `admin:*` scope or use of the master admin key:

<CodeBlock language="bash">
{`curl -X GET https://api.aura-llm.dev/v1/api-keys \\
  -H "Authorization: Bearer \$AURA_ADMIN_KEY"`}
</CodeBlock>

---

## API Keys

### List API Keys

Retrieve all API keys for your organization.

<CodeBlock language="http">
{`GET /v1/api-keys`}
</CodeBlock>

**Query Parameters**

| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by status: `active`, `revoked`, `expired` |
| `org_id` | string | Filter by organization |
| `limit` | integer | Max results (default: 100) |
| `offset` | integer | Pagination offset |

**Response**

<CodeBlock language="json">
{`{
  "data": [

      "project_id": null,

        "requests_per_minute": 100,

### Create API Key

Create a new API key with specified scopes and limits.

<CodeBlock language="http">
{`POST /v1/api-keys`}
</CodeBlock>

**Request Body**

<CodeBlock language="json">
{`{
  "name": "Mobile App Key",
  "scopes": ["responses:create"],
  "org_id": "org_xyz",
  "team_id": "team_123",
  "project_id": "proj_456",
  "rate_limit": {
    "requests_per_minute": 60,
    "tokens_per_day": 100000
  },
  "expires_at": "2027-01-01T00:00:00Z",
  "metadata": {
    "environment": "production",
    "app": "mobile-ios"
  }
}`}
</CodeBlock>

**Parameters**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Human-readable name |
| `scopes` | string[] | Yes | Permission scopes |
| `org_id` | string | No | Organization to associate |
| `team_id` | string | No | Team to associate |
| `project_id` | string | No | Project to associate |
| `rate_limit` | object | No | Rate limit configuration |
| `expires_at` | string | No | Expiration datetime (ISO 8601) |
| `metadata` | object | No | Custom key-value metadata |

**Available Scopes**

| Scope | Description |
|-------|-------------|
| `responses:create` | Create new responses |
| `responses:read` | Read response history |
| `conversations:read` | Read conversations |
| `conversations:write` | Create/delete conversations |
| `usage:read` | Read usage statistics |
| `admin:keys` | Manage API keys |
| `admin:orgs` | Manage organizations |
| `admin:*` | Full admin access |
| `*` | All permissions |

**Response**

<CodeBlock language="json">
{`{
  "id": "key_new789",
  "name": "Mobile App Key",
  "key": "aura_prod_sk_live_abc123xyz...",
  "prefix": "aura_prod_",
  "scopes": ["responses:create"],
  "status": "active",
  "created_at": "2026-01-27T12:00:00Z"
}`}
</CodeBlock>

<Callout type="danger">
The full API key is only returned once at creation. Store it securelyâ€”it cannot be retrieved later.
</Callout>

### Get API Key

Retrieve details for a specific API key.

<CodeBlock language="http">
{`GET /v1/api-keys/{key_id}`}
</CodeBlock>

---

### Update API Key

Update an existing API key's name, scopes, or limits.

<CodeBlock language="http">
{`PATCH /v1/api-keys/{key_id}`}
</CodeBlock>

<CodeBlock language="json">
{`{
  "name": "Updated Name",
  "scopes": ["responses:create", "conversations:read"],
  "rate_limit": {
    "requests_per_minute": 120
  }
}`}
</CodeBlock>

---

### Revoke API Key

Revoke an API key immediately. This action cannot be undone.

<CodeBlock language="http">
{`DELETE /v1/api-keys/{key_id}`}
</CodeBlock>

**Response**

<CodeBlock language="json">
{`{
  "id": "key_abc123",
  "status": "revoked",
  "revoked_at": "2026-01-27T12:00:00Z"
}`}
</CodeBlock>

## Provider Credentials

Manage encrypted provider API keys.

### List Credentials

<CodeBlock language="http">
{`GET /v1/credentials`}
</CodeBlock>

**Response**

<CodeBlock language="json">
{`{
  "data": [
    {
      "id": "cred_abc123",
      "provider": "openai",
      "name": "Production OpenAI",
      "status": "active",
      "org_id": "org_xyz",
      "created_at": "2026-01-01T00:00:00Z",
      "last_used_at": "2026-01-27T08:00:00Z"
    }
  ]
}`}
</CodeBlock>

<Callout type="info">
API keys are encrypted at rest and never returned in responses.
</Callout>

---

### Create Credential

<CodeBlock language="http">
{`POST /v1/credentials`}
</CodeBlock>

<CodeBlock language="json">
{`{
  "provider": "openai",
  "name": "Production OpenAI",
  "api_key": "sk-...",
  "org_id": "org_xyz"
}`}
</CodeBlock>

**Supported Providers**

| Provider | Value |
|----------|-------|
| OpenAI | `openai` |
| Anthropic | `anthropic` |
| Google | `google` |

## Error Responses

Admin API errors follow the standard error format:

<CodeBlock language="json">
{`{
  "error": {
    "code": "forbidden",
    "message": "API key does not have admin:keys scope",
    "request_id": "req_abc123"
  }
}`}
</CodeBlock>

| Code | Status | Description |
|------|--------|-------------|
| `unauthorized` | 401 | Missing or invalid API key |
| `forbidden` | 403 | Insufficient permissions |
| `not_found` | 404 | Resource not found |
| `conflict` | 409 | Resource already exists |
| `validation_error` | 422 | Invalid request body |
