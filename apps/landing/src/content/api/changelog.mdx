# API Changelog

Track all changes to the Aura API. We follow semantic versioning and announce breaking changes with advance notice.

<Callout type="info">
Subscribe to the [GitHub releases](https://github.com/UmaiTech/aura-llm-gateway/releases) to get notified of new versions.
</Callout>

---

## January 2026

### v0.2.6 - January 27, 2026

**New Features**
- Added MDX support for interactive documentation
- New `/docs/guides/existing-sdks` guide for LangChain, LlamaIndex integration
- Enhanced rate limiting documentation with retry strategies

**Improvements**
- Improved error messages for invalid API keys
- Better streaming performance for long responses

---

### v0.2.5 - January 20, 2026

**New Features**
- End-user cost tracking with `user` field in requests
- Per-user rate limiting support
- Token budget enforcement per organization

**API Changes**
- Added `user` field to request body (optional)
- Added `aura.end_user` to response metadata when user tracking enabled

<CodeBlock language="json" title="New user field in requests">
{`{
  "model": "gpt-4.5",
  "input": [...],
  "user": "user_12345"
}`}
</CodeBlock>

---

### v0.2.4 - January 15, 2026

**New Features**
- Provider credential encryption with AES-256-GCM
- Multi-credential support per provider
- Credential rotation without downtime

**Security**
- Master key-based envelope encryption
- Secure credential storage in PostgreSQL

### v0.2.0 - December 20, 2025

**Major Release**

This release introduces the Open Responses API specification and multi-provider architecture.

**Breaking Changes**
- Migrated from OpenAI Chat Completions format to Open Responses API
- Response structure changed (see migration guide)
- Streaming events renamed for semantic clarity

<Callout type="warning">
**Migration Required**: If upgrading from v0.1.x, see the [Migration Guide](/docs/guides/migration).
</Callout>

**New Response Format**
<CodeBlock language="json">
{`{
  "id": "resp_abc123",
  "status": "completed",
  "output": [
    {
      "type": "message",
      "role": "assistant",
      "content": "Hello!"
    }
  ],
  "usage": { ... },
  "aura": { ... }
}`}
</CodeBlock>

**Previous Format (v0.1.x)**
<CodeBlock language="json">
{`{
  "id": "chatcmpl-abc123",
  "choices": [
    {
      "message": {
        "role": "assistant",
        "content": "Hello!"
      }
    }
  ]
}`}
</CodeBlock>

## Deprecation Policy

We provide **6 months notice** before removing deprecated features. Deprecated items are marked in responses with the `X-Aura-Deprecated` header.

### Currently Deprecated

| Feature | Deprecated | Removal Date | Replacement |
|---------|------------|--------------|-------------|
| `choices` response format | v0.2.0 | June 2026 | `output` array |
| `messages` input format | v0.2.0 | June 2026 | `input` items |

### Sunset Features

None currently scheduled.

---

## Versioning Policy

Aura follows [Semantic Versioning](https://semver.org/):

- **Major (X.0.0)**: Breaking changes requiring migration
- **Minor (0.X.0)**: New features, backwards compatible
- **Patch (0.0.X)**: Bug fixes, no API changes
