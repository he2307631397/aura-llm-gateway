# Prompt Compression

Aura can automatically compress prompts to reduce token usage and costs while maintaining response quality.

<Callout type="info">
Prompt compression is an Aura extension to the Open Responses API. It applies intelligent transformations to reduce input tokens without losing semantic meaning.
</Callout>

## Compression Strategies

| Strategy | Savings | Best For |
|----------|---------|----------|
| **JSON Minification** | 15-30% | Structured data |
| **TOON** | 40-60% | Uniform arrays |
| **YAML** | 10-25% | Nested objects |
| **AISP** | Clarity boost | Rules and logic |

## Quick Start

Enable compression with auto-selection:

<CodeBlock language="json" title="Request with compression">
{`{
  "model": "gpt-5.4-mini",
  "input": [
    {
      "type": "message",
      "role": "user",
      "content": "Summarize these results"
    }
  ],
  "context": {
    "documents": [
      {"id": 1, "title": "Doc 1", "score": 0.95},
      {"id": 2, "title": "Doc 2", "score": 0.87},
      {"id": 3, "title": "Doc 3", "score": 0.82}
    ]
  },
  "compression": {
    "enabled": true,
    "auto_select": true
  }
}`}
</CodeBlock>

The response includes compression metadata:

<CodeBlock language="json" title="Response with compression stats">
{`{
  "usage": {
    "input_tokens": 850,
    "output_tokens": 200,
    "compression": {
      "original_tokens": 2400,
      "compressed_tokens": 850,
      "ratio": 0.35,
      "strategies": ["toon"],
      "latency_ms": 8
    }
  }
}`}
</CodeBlock>

## Configuration Options

<CodeBlock language="json" title="Full compression config">
{`{
  "compression": {
    "enabled": true,
    "data_format": "auto",
    "semantic_format": "natural",
    "auto_select": true,
    "target_ratio": null,
    "token_budget": null,
    "token_cleanup": true,
    "minify_json": true
  }
}`}
</CodeBlock>

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `enabled` | boolean | `false` | Enable compression |
| `data_format` | string | `"json_compact"` | Format for structured data |
| `semantic_format` | string | `"natural"` | Format for instructions |
| `auto_select` | boolean | `false` | Auto-select best format per content |
| `target_ratio` | number | null | Target compression ratio (0.0-1.0) |
| `token_budget` | number | null | Maximum tokens after compression |

## TOON Format

TOON (Token-Oriented Object Notation) achieves **40-60% token savings** on uniform arrays by declaring field names once.

<CodeBlock language="text" title="JSON vs TOON">
{`// JSON (many tokens)
[
  {"id": 1, "name": "Alice", "role": "admin"},
  {"id": 2, "name": "Bob", "role": "user"}
]

// TOON (fewer tokens)
[2]{id,name,role}:
  1,Alice,admin
  2,Bob,user`}
</CodeBlock>

<Callout type="tip">
TOON is especially effective for RAG retrieval results, user lists, logs, and any array of similar objects.
</Callout>

<Expandable title="TOON Configuration">
<CodeBlock language="json">
{`{
  "compression": {
    "toon": {
      "min_array_size": 2,
      "min_fields": 2,
      "max_depth": 3
    }
  }
}`}
</CodeBlock>

| Field | Default | Description |
|-------|---------|-------------|
| `min_array_size` | 2 | Minimum array size for TOON encoding |
| `min_fields` | 2 | Minimum fields per object |
| `max_depth` | 3 | Maximum nesting depth |
</Expandable>

## AISP (AI Symbolic Protocol)

AISP converts natural language rules to formal mathematical notation, reducing ambiguity from **40-65% to under 2%**.

<CodeBlock language="text" title="Natural Language vs AISP">
{`# Natural Language (ambiguous)
"For all users, if they are an admin, allow access"

# AISP (unambiguous)
∀u∈Users: admin(u) ⇒ allow(u)`}
</CodeBlock>

### Common AISP Symbols

| Symbol | Meaning | Example |
|--------|---------|---------|
| `∀` | For all | `∀x∈Set` |
| `∃` | There exists | `∃x: valid(x)` |
| `⇒` | Implies | `A ⇒ B` |
| `∧` | Logical AND | `A ∧ B` |
| `∨` | Logical OR | `A ∨ B` |
| `¬` | Logical NOT | `¬A` |
| `∈` | Element of | `x ∈ Set` |
| `≜` | Defined as | `x ≜ 5` |

<Callout type="info">
AISP is based on the [AI Symbolic Protocol](https://github.com/bar181/aisp-open-core) specification. It uses a standardized set of mathematical symbols understood by modern LLMs.
</Callout>

<Expandable title="AISP Configuration">
<CodeBlock language="json">
{`{
  "compression": {
    "semantic_format": "aisp",
    "aisp": {
      "symbol_set": "core",
      "convert_rules": true,
      "convert_definitions": true,
      "convert_quantifiers": true
    }
  }
}`}
</CodeBlock>

**Symbol sets:**
- `core` - ~50 most common symbols (default)
- `standard` - ~150 practical symbols
- `full` - All 512 symbols
</Expandable>

## YAML Conversion

YAML uses fewer tokens than JSON by eliminating quotes, commas, and brackets:

<CodeBlock language="yaml" title="YAML output">
{`user:
  name: Alice
  email: alice@example.com
  roles: [admin, user]
  active: true`}
</CodeBlock>

<Callout type="tip">
YAML is best for deeply nested configuration objects. For arrays, TOON is more efficient.
</Callout>

## Examples

### Compressing System Instructions

<CodeBlock language="json" title="System prompt with AISP">
{`{
  "model": "gpt-5.4-mini",
  "instructions": "For all user requests, if the user is authenticated then allow access. If not authenticated, reject with error.",
  "input": [...],
  "compression": {
    "enabled": true,
    "semantic_format": "aisp"
  }
}`}
</CodeBlock>

The instructions will be converted to:

<CodeBlock language="text" title="Compressed instructions">
{`∀r∈Requests: authenticated(r.user) ⇒ allow(r)
∀r∈Requests: ¬authenticated(r.user) ⇒ reject(r, error)`}
</CodeBlock>

### Batch Data Processing

<CodeBlock language="json" title="RAG context with TOON">
{`{
  "model": "gpt-5.4-mini",
  "input": [
    {
      "type": "message",
      "role": "user",
      "content": "Summarize these search results"
    }
  ],
  "context": {
    "documents": [
      {"id": 1, "title": "Result 1", "score": 0.95, "excerpt": "..."},
      {"id": 2, "title": "Result 2", "score": 0.87, "excerpt": "..."},
      {"id": 3, "title": "Result 3", "score": 0.82, "excerpt": "..."}
    ]
  },
  "compression": {
    "enabled": true,
    "data_format": "toon"
  }
}`}
</CodeBlock>

## Cost Impact

Compression reduces input tokens, directly lowering costs:

| Compression | Input Tokens | Cost (GPT-4o) | Savings |
|-------------|--------------|---------------|---------|
| None | 2,400 | $0.012 | - |
| TOON (35% ratio) | 850 | $0.004 | $0.008 |
| JSON Minify (75% ratio) | 1,800 | $0.009 | $0.003 |

<Callout type="warning">
Compression latency is typically 1-10ms. For very latency-sensitive applications, you may want to disable compression or use only JSON minification.
</Callout>

## Best Practices

1. **Use auto-select** - Let Aura choose the best format automatically

2. **TOON for batch data** - Arrays of similar objects (RAG results, user lists, logs)

3. **AISP for complex rules** - System prompts with conditional logic, validation rules

4. **Monitor the metadata** - Check `compression` stats in responses to measure actual savings

5. **Combine with caching** - Cached compressed prompts provide maximum cost reduction
